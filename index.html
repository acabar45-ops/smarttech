2 · 탭별 기능

2.1 공정별 추천

9개 공정 카테고리(코팅, 이차전지, 진공로, 가스실린더, 진공이중배관, 동결건조, OLED, 진공오븐, 연구/분석)를 타일로 선택합니다. 카드 클릭 시 해당 공정의 진공도·펌프 유형·핵심 포인트·추천 모델군이 표시되고, 모델 키워드(예: nXDS, GXS) 를 기반으로 914개 전체 카탈로그에서 부분일치 최대 25건을 추천합니다. 담기 + 로 장바구니에 추가되며, 단가 표시는 현재 선택된 등급가를 따릅니다.

2.2 품목 검색

품명 또는 Part No 부분일치 검색. 시트 필터(IV/SV/VTS)로 세트를 좁힐 수 있습니다. 결과는 최대 100개. 각 행에 Dealer / OEM / End User 3개 단가가 병렬 표시되어 등급 비교가 즉시 가능합니다.

2.3 견적서 작성

견적 작성의 중심 화면. 아래 순서로 구성됩니다.

  1. 등급 선택 + 등급 배수 입력 — 딜러/OEM/최종사용자 3등급 중 하나. 각 등급 라벨 우측의 % 입력칸이 현재 등급별 배수이며, 기본 100%. 50~150% 범위에서 0.1% 단위 조정. (설정 탭에서 동일 값을 관리합니다.)
  2. 견적 미세 조정 — 이번 견적에만 적용되는 전체 가격 조정 슬라이더(50~150%). 장바구니 저장·불러오기 시 함께 보존됩니다.
  3. 명함 스캔(OCR) — 이미지 업로드 → Claude Vision 명함 파서 에이전트 실행(키 미설정 시 Tesseract). 회사·담당자·전화·이메일이 폼에 자동 반영.
  4. 고객 정보 폼 — 회사명/담당자/연락처/이메일/견적번호/납기. 견적번호 기본값은 YYMMDD_SR01.
  5. 견적 품목 / 합계 — 수량·단가 직접 수정 가능. 합계는 실시간 재계산.
  6. 고객 저장 — 현재 폼 내용을 CRM(고객 관리 탭)에 저장. 동일 회사+이메일 조합은 덮어쓰기.

2.4 미리보기 · 출력

미리보기 → 를 누르면 견적서가 A4 포맷으로 렌더링되고, 동시에 발행 내역에 자동 아카이브됩니다. 브라우저 인쇄 → PDF로 저장으로 파일 추출. 텍스트 복사 는 메일 본문용 평문 추출입니다.

2.5 고객 관리 (CRM)

mk.clients.v1 에 저장된 고객 레코드를 검색·불러오기·삭제. JSON 내보내기/가져오기로 기기 간 이전 가능. 불러오기는 해당 고객의 회사·담당자·연락처·이메일·등급을 자동으로 견적서 작성 폼에 채웁니다.

2.6 발행 내역 (Archive)

미리보기 단계에서 자동 저장된 견적서 레코드 목록. 견적번호·발행일·수신·등급·공급가액·총액 열람. 불러오기 로 과거 견적 전체(장바구니 품목·단가·등급·미세조정·고객정보)를 복원. 동일 견적번호 재발행 시 덮어쓰기. JSON 내보내기 지원.

2.7 메일 자동견적 (2-Agent)

고객사 문의 메일 전체를 붙여넣고 에이전트 분석 시작 클릭 → 두 에이전트가 순차 실행(상세 로직은 §4). 제안 결과는 카드로 표시되며 적용 시 고객정보/등급/장바구니가 한 번에 설정되고 견적서 작성 탭으로 이동합니다.

2.8 설정

등급별 % 배수 일괄 관리(잠금/수정 모드), 초기화, Anthropic API 키 저장(password 필드, localStorage), 모델 선택(기본 claude-haiku-4-5 · 고정밀 claude-sonnet-4-6), 엔진 상태 표시.

3 · 가격 계산 공식

모든 단가는 엑셀 원본값(등급별 3개 컬럼)을 기준으로 아래 순서로 계산됩니다. 계산 함수는 js/script.js 의 getP(p).

STEP 1 기준가 = p.dealer | p.oem | p.endUser (현재 등급 컬럼)
STEP 2 등급 조정가 = STEP1 × MK_SETTINGS.multipliers[grade] / 100
STEP 3 표시 단가 = STEP2 × MK_QUOTE_ADJUST / 100 → Math.round()
STEP 4 라인 금액 = 표시 단가 × 수량  (사용자가 수동으로 customPrice 를 입력한 행은 STEP 2–3 을 건너뛰고 입력값이 그대로 사용됨)
STEP 5 공급가액 = Σ 라인 금액   ·   부가세 = round(공급가액 × 0.1)   ·   총액 = 공급가액 + 부가세

핵심: 등급 배수(설정)와 견적 미세조정은 곱연산으로 누적됩니다. 예) Dealer 기준가 100,000원 × 배수 95% × 조정 98% = 93,100 원. 배수는 설정 탭에서 저장되어 모든 견적에 지속 적용되고, 조정은 현재 견적에만 적용됩니다.

4 · AI 에이전트 로직

모든 AI 호출은 Anthropic Messages API의 tools + tool_choice 메커니즘으로 구조화된 JSON 도구 호출을 강제합니다. 이는 자유 텍스트 응답보다 파싱 실패율을 크게 낮춥니다.

4.1 명함 판독 에이전트

파일: js/extras.js · 함수 parseWithClaude().

  • 입력: 이미지 파일(Base64) + 판독 규칙 프롬프트.
  • 도구: extract_business_card · 스키마 필드 — company, company_en, contact, title, department, mobile, office, fax, email, address, website, memo, confidence.
  • 규칙 요약: 법인격(㈜, Co., Ltd.) 원문 보존; 전화번호 010-0000-0000 포맷; 국가코드(+82) 0 로 치환; 01X 계열은 mobile, 그 외는 office; 직급과 부서 분리; 명함에 없는 필드는 null (환각 금지); confidence 자체 평가.
  • Fallback: API 키 미설정 시 Tesseract.js 로 한/영 OCR 후 정규식 파싱(회사는 법인격 키워드, 담당자는 직급 키워드, 전화는 휴대 형식 우선).
  • UI: 반환된 모든 필드를 표로 표시 + 신뢰도 · 모델 · 토큰 사용량. 핵심 4개(회사·담당자·전화·이메일)만 폼에 자동 반영.

4.2 메일 자동견적 · Agent 1 (등급 분류 + 고객정보)

파일: js/email-agent.js · 함수 callTool("classify_grade", ...).

  • 역할: 메일 본문에서 거래 등급(dealer|oem|enduser)을 판정하고, 동시에 회사·담당자·직급·전화·이메일·주소·문의요지를 추출.
  • 판정 기준:
    • 딜러: 재판매/총판/대리점/판매처/리셀러(reseller/distributor), 자신을 딜러로 명시, 다수 최종사용자에게 납품 목적
    • OEM: 자사 장비에 내장/통합(integration)·BOM·생산라인 투입, 반복·연속 수요 시그널
    • 최종사용자: 대학·연구소·공장 공정 직접 운영, 소수 자체 사용, 구체 공정 묘사
    • 시그널이 약하면 enduser 로 보수적 기본값, confidence 하향
  • 출력: { grade, confidence, rationale, signals[], customer{...}, subject_summary }. signals 는 판정 근거가 된 메일 원문 발췌 3~6개.

4.3 카탈로그 사전 필터 (클라이언트 측)

Agent 2 호출 전, 914개 전체를 LLM 에 넘기지 않기 위해 브라우저에서 후보를 최대 80개로 축약합니다. 함수 prefilterCatalog().

  1. 메일 텍스트에서 토큰화 ([a-z][a-z0-9\-]{2,}) → 영문 모델명 후보군 수집.
  2. 공정 키워드 매핑표(코팅/이차전지/진공로/동결건조 등 9종)로 매칭되면 관련 모델 시리즈(nXDS, GXS, nEXT 등)를 추가 토큰으로 확장.
  3. 각 제품의 partNo + description 에 토큰이 포함되면 점수화. 앞부분 일치(≤20 자) = +3, 그 외 +1, 정확 Part No 매칭 = +10.
  4. 점수 > 0 만 필터링 → 내림차순 정렬 → 상위 80개만 Agent 2 에 전달.

4.4 메일 자동견적 · Agent 2 (제품 선정)

함수 callTool("select_products", ...). 입력에 Agent 1 의 등급 + 메일 원문 + 사전필터 후보(JSON [{p, d, s}]) 를 함께 제공.

  • 규칙: 카탈로그 외 Part No 생성 금지(환각 방지). 모델명 명시 시 최우선 매칭. 수량 명시가 없으면 1. 공정/용도만 언급되면 펌프·부스터·액세서리 조합 5~8개 추천. 확신 부족 시 notes 에 확인 필요 사항 기재.
  • 출력: { items: [{ partNo, qty, rationale }], confidence, notes }.
  • 검증: 클라이언트에서 각 partNo 를 ALL_PRODUCTS 와 재매칭. 존재하지 않는 항목은 제외하고 개수를 경고로 표시.

4.5 적용 단계

적용 클릭 시 applyEmailProposal() 실행:

  1. 고객정보 4개 필드 폼 반영 + 담당자 = 이름 + 직급.
  2. setGrade() 로 등급 설정 → 등급 배수 자동 적용.
  3. 장바구니가 비어있으면 그대로 추가, 품목이 있으면 "교체 / 추가" 확인 대화상자.
  4. 기존 장바구니에 동일 Part No 가 있으면 수량 가산, 없으면 신규 행.
  5. switchTab("quote") 로 이동 → 사용자 검토 후 미리보기 → 로 진행.
